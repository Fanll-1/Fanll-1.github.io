个人博客使用 ——如何上传一篇自己的文章

Matery Hexo
个人博客
发布日期:   2021-09-30
文章字数:   703

了解

  • 在写博客文章时，使用的时markdown语法，通过一些简单的标记语法让文本巨有一定的格式。
  • 编写工具：Typora或 Vscode + Markdown插件。我比较喜欢使用后者进行编写。
  • 点击这里了解更多关于markdown的编写规则，轻松上手。

新建页面

  • 一篇新的博客文章就是一个md文件，需要通过命令来创建 hexo new title。例如 hexo new "我的第一篇文章"。

  • 创建md文件时，可以指定布局。布局有三种：post(文章),draft(草稿),page(页面)

    $ hexo new [layout] title
$ hexo new post "文章"
$ hexo new page "页面"
$ hexo new draft "草稿"

    如果不指定布局，将默认生成post布局

  • draft这种布局在建立时会被保存到 source/_drafts 文件夹中，但不会显示在页面上，如果我们不想某一篇文章显示在页面上，那么就可以把它移动到_drafts文件夹中。
    我们可在启动服务器时加上 --draft 参数来查看草稿。

    $ hexo server --draft

    还可以在站点配置文件中把 render_drafts 参数设为 true 来预览草稿。
    我们可以通过 publish 命令将草稿发布文章或者页面，它将会被移动到指定的文件夹。

    $ hexo publish [layout] title

    内容编写

    Front-matter区域

    （创建md文档后自动生成的内容）

    ---
title: 文章 //对应的是创建命令时输入的title名字
date: 2021/9/30 12:14:35
---

    预定义参数

    layout：布局，默认为true，如果你不想你的文章被处理，可以设置为false
    title ：标题，标题会显示在最上方居中位置
    date：建立日期，如果不指定则为默认值-文件创建日期，可以自定义。
    update：更新日期，如果不指定则为默认值-文件修改后重新生成静态文件的日期。
    comments：是否开启文章的评论功能 默认值为true
    tags：标签（适用雨post页面）
    categoreies：分类（适用雨post页面）
    permalink：覆盖文章网址
    keywords：仅用于 meta 标签和 Open Graph 的关键词（不推荐使用）

发表文章

进入博客根目录，依次输入git命令

$ hexo clean  //清除缓存
$ hexo g      //生成静态文件 hexo generate
$ hexo s      //启动本地服务器 hexo server
$ hexo d      //将网站部署到服务器上，网站内容就会更新了 hexo deploy

我们也可以这样写

$ hexo clean && hexo g && hexo s
$ hexo d
